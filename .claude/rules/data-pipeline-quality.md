# Rule: 데이터 파이프라인 품질 기준

## 데이터 유형 분류

| 유형 | 예시 | 처리 방식 |
|------|------|-----------|
| 정적 | 호텔명, 주소, 시설 요약, 객실 타입 | DB 적재, CSV 업로드 허용 |
| 동적 | 가격, 혜택, 환불규정, 조식 포함 여부 | API/피드 우선, 보조 크롤링 |

## data_coverage_score (커버리지 점수)

각 호텔의 데이터 품질을 0~100점으로 측정.

| 항목 | 배점 |
|------|------|
| 사진 ≥ 5장 | 20점 |
| 어메니티 ≥ 10개 | 15점 |
| 위치 설명 있음 | 15점 |
| 후기 요약 있음 | 15점 |
| 객실 타입 설명 있음 | 10점 |
| 교통/동선 정보 있음 | 10점 |
| 가격대 정보 있음 | 10점 |
| 연락처/체크인 정보 | 5점 |

**60점 미만:** 단독 리뷰형 글 발행 금지, 비교표 카드로만 제한
**40점 미만:** 모든 콘텐츠 제외, 데이터 보강 후 재검토

커버리지 점수는 `state/coverage/` 에 기록.

## CSV/엑셀 업로드 처리 기준

1. 업로드 후 필수 필드 존재 여부 검증
2. 중복 호텔 ID 감지 및 병합/거부
3. 이상값(가격 0원, 이름 없음 등) 플래그 처리
4. 정규화 완료 후 `data/processed/`에 저장
5. 원본은 `data/hotels/`에 보존

## 데이터 소스 우선순위

1순위: 공식 API (아고다 API, 파트너 피드)
2순위: 공식 콘텐츠 파일 (파트너 제공 CSV/JSON)
3순위: 보조 크롤링/스크래핑 (불가피한 경우만, 정책 준수)

## 크롤링 사용 시 주의사항

- robots.txt 준수 필수
- 요청 간격 최소 2초 이상
- User-Agent 명시
- 법적 이슈가 불명확한 경우 사용 금지

## 데이터 만료 관리

- 동적 데이터(가격/혜택)는 캐시 유효기간 24시간
- 만료된 데이터는 "확인 필요" 표시, 자동 노출 금지
- 글 발행 시 데이터 수집 날짜 기록
