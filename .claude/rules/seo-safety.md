# Rule: SEO 안전 원칙

## 핵심 원칙
단일 도메인 권위 축적. 품질 콘텐츠만 발행. 속임수 없음.

## 절대 금지 패턴

### 도어웨이 페이지
- 동일 콘텐츠를 도시명/지역명만 바꿔 복제 → 즉시 차단
- "서울 호텔 추천", "부산 호텔 추천" 등 구조만 같고 내용 없는 페이지 → 금지
- 검색 크롤러용으로만 최적화된 페이지 → 금지

### 키워드 스터핑
- 제목/소제목에 동일 키워드 반복 삽입 → 금지
- 키워드 밀도를 인위적으로 높이는 행위 → 금지
- 숨겨진 텍스트/링크 → 금지

### 중복 콘텐츠
- 동일 글의 언어만 기계 번역한 버전을 별도 URL로 운영 → 금지
- canonical 없이 파라미터 URL 다수 생성 → 금지

## 다국어 SEO 요구사항

- URL 구조: `tripprice.com/ko/`, `/en/`, `/ja/` (서브디렉토리)
- 각 언어 페이지에 hreflang 태그 필수
- 번역본은 단순 기계번역 금지 → 현지화 편집 필수
- canonical은 각 언어별 자기 자신을 가리킴

## 콘텐츠 차별화 요구사항

새 글 발행 전 반드시 확인:
1. 동일 키워드를 이미 다루는 내부 글이 있는가?
2. 있다면 → 신규 발행 대신 기존 글 개선을 우선 검토
3. 차별점(관점/페르소나/동선/예산대)이 없으면 발행 금지

## 구조화 데이터 권장

- Hotel schema: 호텔 정보 페이지
- FAQPage schema: FAQ 포함 글
- Article schema: 가이드형 글
- BreadcrumbList: 모든 내부 페이지

## slug 네이밍 규칙

- 소문자 + 하이픈 구분
- 키워드 포함, 간결하게 (3~5 단어)
- 연도 포함 여부: 시의성 있는 글만 (예: 2025-seoul-hotel-guide)
- 특수문자, 언더스코어 금지
