# Rule: WordPress 출력 형식 기준

## 필수 필드 (wp-post-schema.json 참고)

| 필드 | 요구사항 |
|------|---------|
| post_title | 60자 이하, 검색 의도 키워드 포함 |
| slug | 소문자-하이픈, 3~5단어 |
| meta_description | 120~155자, CTA 문구 포함 |
| excerpt | 2~3문장, 독자 타겟 + 핵심 가치 |
| post_status | 항상 "draft" (사람이 검토 후 publish) |
| lang | ko / en / ja 명시 |
| hreflang_links | 연결된 다국어 버전 URL 배열 |
| affiliate_links | 삽입된 아고다 링크 배열 (추적용) |

## 본문 구조 (H 태그 순서)

```
H1: 글 제목 (1개만, 검색 의도 키워드 포함)
  H2: 빠른 결론 요약
  H2: 이 글이 필요한 사람
  H2: 선택 기준
  H2: 호텔 비교 / 추천
    H3: [호텔명] — [한 줄 포지셔닝]
      (추천 대상, 장점, 단점, 위치·동선, 주의점)
  H2: 자주 묻는 질문 (FAQ)
  H2: 예약 전 최종 체크리스트 (선택)
```

## 이미지 삽입 기준

- 대표 이미지(featured image): 반드시 1개, WebP, 1200×630px 권장
- 본문 이미지: 호텔당 최소 1개, alt 텍스트 필수
- alt 텍스트: "[호텔명] [특징] [위치]" 형식, 키워드 반복 금지
- 파일명: hotel-name-feature-location.webp (소문자 하이픈)

## CTA 블록 배치 규칙

- 위치: 각 호텔 섹션 끝 + 글 하단
- 형식: 버튼 또는 박스형 (인라인 텍스트 링크 과다 사용 금지)
- 문구 예시: "아고다에서 현재 가격 확인하기 →"
- 개수: 글당 최대 4개 CTA (과도한 삽입 금지)

## 내부 링크 규칙

- 최소 2개, 최대 5개
- 허브 글(도시 가이드, 지역 가이드)으로 연결 우선
- 앵커 텍스트: 목적지 글의 키워드 반영, "여기 클릭" 금지
