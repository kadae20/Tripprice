# Rule: 이미지 정책 및 처리 기준

## 이미지 소스 우선순위

1순위: 파트너 공식 제공 이미지 (API/피드)
2순위: 자체 제작 (지도, 인포그래픽, 동선 이미지, 요약 카드)
3순위: 라이선스 명확한 스톡 이미지 (Unsplash 등)
4순위: 크롤링 수집 (파트너 정책 명확히 허용된 경우만)

## 처리 파이프라인

```
수집(raw/) → 리사이즈/WebP변환 → 워터마크(선택) → alt생성 → WP업로드
```

### 1단계: 수집
- 원본 이미지 → `assets/raw/[hotel-id]/`에 저장
- 소스 출처, 수집 날짜 메타데이터 기록

### 2단계: 최적화
- WebP 변환 필수
- 대표 이미지: 1200×630px
- 본문 이미지: 최대 너비 1080px, 최대 용량 200KB
- 처리 결과 → `assets/processed/[hotel-id]/`에 저장

### 3단계: 워터마크 (조건부)
- 파트너 정책이 허용하는 경우만 적용
- 위치: 모서리 (우하단 권장), 크기 작게 (이미지 너비의 10% 이하)
- 정보 전달을 방해하는 워터마크 금지
- 불명확한 경우: 원본 최소 가공, 대표 이미지는 자체 제작으로 대체

### 4단계: alt 텍스트 자동 생성 규칙
- 형식: `[호텔명] [특징 설명] [위치]`
- 예시: "그랜드 하얏트 서울 루프탑 수영장 이태원"
- 금지: 호텔명만 반복 (`그랜드 하얏트 서울 그랜드 하얏트 서울`)
- 금지: "이미지", "사진", "photo" 등 무의미한 단어
- 글자 수: 50~100자 권장

### 5단계: WordPress 업로드
- 파일명: `hotel-name-feature.webp` (소문자 하이픈)
- 미디어 라이브러리에 호텔 ID 태그 부착
- 본문 삽입 시 srcset 설정 권장

## 사진 부족 호텔 대응 전략

data_coverage_score에서 사진 항목 미달인 경우:
- 단독 리뷰형 글 → 지역/동선/예산 가이드형 글로 전환
- 호텔 단독 이미지 대신: 지도, 주변 랜드마크, 동선 인포그래픽 사용
- 비교표 카드 형식으로만 노출 허용

## 권리/정책 준수

- 파트너 이미지 가공 전 파트너 약관 확인 필수
- 무단 가공/워터마크 삽입 후 분쟁 발생 시 즉시 원본으로 복구
- 이미지 출처는 메타데이터로 기록, 언제든 추적 가능하게 유지
