# Skill: compare-hotels

## 역할
여러 호텔을 구조화된 비교표로 정리하고, 상황별 추천 논리를 제공한다.

## 호출 방식
수동 호출 — write-hotel-article 내부에서 서브 작업으로도 호출 가능.

## 허용 도구
- 파일 읽기/쓰기
- data/processed/ 접근

## 입력
- 호텔 ID 목록 (2~6개 권장)
- 비교 기준 (예: 위치, 가격대, 조식, 수영장 등)
- 타겟 페르소나

## 출력
content-templates/hotel-comparison-table.md 구조를 따른 비교표 + 추천 논리.

```markdown
## 한눈에 비교

| 기준 | [호텔 A] | [호텔 B] | [호텔 C] |
|------|----------|----------|----------|
| 위치 | ... | ... | ... |
| 가격대 | ... | ... | ... |
| 조식 | ... | ... | ... |
| 추천 대상 | ... | ... | ... |

## 상황별 추천
- 가족 여행: [호텔명] — 이유
- 커플: [호텔명] — 이유
- 출장: [호텔명] — 이유
```

## 품질 게이트
- 모든 호텔이 동점/동급 추천이면 → 차별점 재분석 필요
- data_coverage_score 40점 미만 호텔은 비교표에서 제외
