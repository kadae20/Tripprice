# Skill: plan-article

## 역할
새 글의 검색 의도, 타겟 독자, 차별점을 정의하고 발행 적합성을 판단한다.

## 호출 방식
수동 호출 — 새 글 작성 전 반드시 실행.

## 허용 도구
- 파일 읽기/쓰기 (state/, data/)
- 웹 검색 (경쟁 글 현황 파악)
- 파일 생성 (브리프 초안)

## 입력
- 주제 또는 키워드 (예: "도쿄 신주쿠 호텔 추천 가족 여행")
- 타겟 언어 (ko/en/ja)
- 타겟 페르소나 (선택)

## 출력 (state/campaigns/[slug]-plan.md)

```
## 검색 의도
[트랜잭션형 / 정보형 / 탐색형 중 하나 + 설명]

## 타겟 독자
[구체적인 페르소나 1~2줄]

## 차별점
[기존 글 대비 이 글만의 관점/동선/예산/페르소나 차이]

## 경쟁 글 현황
[상위 3개 글 URL + 약점 요약]

## 추천 구조
[H2 목차 초안]

## 발행 적합성
[적합 / 부적합 + 이유]
```

## 품질 게이트
- "발행 적합성: 부적합"이면 build-brief로 진행 금지
- 기존 내부 글과 80% 이상 중복이면 → 신규 발행 대신 refresh-underperforming-post 권장
