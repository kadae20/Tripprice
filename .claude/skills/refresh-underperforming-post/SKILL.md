# Skill: refresh-underperforming-post

## 역할
성과가 낮은 기존 글을 분석하고, 개선 방향을 제안하거나 실제 업데이트한다.

## 호출 방식
수동 호출 — performance-analyst가 부진 글 목록 제공 후 실행.

## 허용 도구
- 파일 읽기/쓰기
- 웹 검색 (현재 SERP, 경쟁글 변화 확인)
- 네트워크 (Search Console 데이터 읽기, 권한 있는 경우)

## 입력
- 기존 글 URL 또는 post_id
- state/campaigns/[slug]-published.json
- 성과 데이터 (클릭수, 노출수, CTR, 평균 순위)

## 분석 항목

1. **검색 의도 변화**: 현재 상위 글과 의도 일치 여부
2. **정보 노후화**: 가격/시설/정책 변경 여부
3. **구조 개선**: 새로운 FAQ/섹션 필요 여부
4. **내부 링크**: 신규 발행글과 연결 누락 여부
5. **경쟁 변화**: 새로운 경쟁글 등장 여부

## 출력 (state/campaigns/[slug]-refresh-plan.md)

```
## 현재 성과 요약
## 주요 문제 진단
## 개선 우선순위 (3가지 이내)
## 개선 후 예상 효과
## 업데이트 범위: [전면 재작성 / 부분 수정 / 정보 업데이트만]
```

## 판단 기준

| 상황 | 권장 액션 |
|------|-----------|
| 순위 11~20위, 구조 양호 | 내부 링크 보강 + 정보 업데이트 |
| 노출 많고 CTR 낮음 | SEO title / meta description 개선 |
| 검색 의도 불일치 | 구조 전면 재작성 |
| 경쟁 과다, 차별점 없음 | 글 통합 또는 방향 전환 |
