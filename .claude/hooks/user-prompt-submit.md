# Hook: UserPromptSubmit

## 목적
사용자 요청에서 스팸/도어웨이/중복 양산 패턴을 감지하고 차단 또는 리디렉트한다.

## 실행 파일
.claude/hooks/scripts/prompt-guard.sh

## 차단 패턴 (감지 시 경고 후 중단)

### 1. 대량 복제 요청
- "도시명만 바꿔서 100개 글 만들어줘"
- "모든 도시에 대해 동일한 구조로 작성"
- "X개 도시를 동시에 발행"

### 2. 키워드 스터핑 요청
- "키워드를 최대한 많이 넣어줘"
- "키워드 밀도를 높여줘"

### 3. 도어웨이 페이지 요청
- "지역명만 바꿔서 발행할 페이지 50개 만들어"
- "같은 내용으로 다른 URL에 올려줘"

### 4. 서브도메인 다발 복제
- "언어별로 서브도메인 새로 만들어"
- "city.tripprice.com 형식으로 50개 도메인"

### 5. Thin affiliate 대량 발행
- "정보 없이 링크만 넣어서 빠르게 발행"
- "호텔 설명 없이 예약 링크만 100개"

## 리디렉트 메시지 형식

```
⚠️ [Tripprice 편집국 경고]
요청이 편집국 절대 규칙에 위배될 수 있습니다.

감지된 패턴: [패턴 설명]

권장 대안:
→ 단일 도메인 품질 글 확장 전략
→ 기존 부진 글 개선 (refresh-underperforming-post)
→ 새 도시 진입 시 keyword-map 먼저 실행

계속 진행하려면 의도를 명확히 설명해주세요.
```

## 허용 패턴 (차단 안 함)
- 단일 글 기획/작성 요청
- 기존 글 개선/업데이트 요청
- 데이터 적재/분석 요청
- 다국어 현지화 (현지화 편집 포함된 경우)
